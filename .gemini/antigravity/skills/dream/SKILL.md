# Dream Mode — Memory Consolidation Skill

You are performing a **memory consolidation cycle**. This is the agent equivalent of "sleeping" — you reorganize your notes to keep long-term context clean, accurate, and compact for better performance in future conversations.

## When to Auto-Trigger Dream Mode

You MUST automatically initiate a dream cycle (without user prompting) when ANY of these conditions are detected during a conversation:

1. **After Major Refactors**: You renamed significant portions of a codebase, changed architecture, or restructured files
2. **Stale References**: You notice MEMORY.md references files, paths, or APIs that no longer exist
3. **Contradictions**: MEMORY.md contains conflicting information (e.g., two different version numbers for the same dependency)
4. **Session Length**: You have performed 10+ tool calls in a single conversation
5. **User Corrections**: The user corrected you on something that contradicts what's in MEMORY.md
6. **Project Completion**: A major task or implementation plan was completed

When auto-triggering, announce: `🌙 Auto-dreaming — consolidating memory...` then proceed with the cycle.

## Dream Cycle Protocol

### Phase 0: Bootstrap (First Dream — No Memory Files Exist)

If ANY of the memory files are missing, create them before proceeding:

**Check for these files:**
- `MEMORY.md` in workspace root → if missing, **create it**
- `CLAUDE.md` in project directories → if missing, **create it**
- `USER_MEMORY.md` in `~/.gemini/antigravity/` → if missing, **create it**

**How to bootstrap each file:**

#### MEMORY.md (workspace root)
Scan the workspace to auto-populate:
1. Run `find {workspace_root} -maxdepth 3 -type f -name "*.java" -o -name "*.json" -o -name "*.gradle" -o -name "*.ts" -o -name "*.tsx" -o -name "package.json" | head -50` to understand project structure
2. Read key config files (package.json, build.gradle, gradle.properties, fabric.mod.json, tsconfig.json)
3. Scan conversation history for user preferences and past decisions
4. Create MEMORY.md with these sections:
   - User Profile (from conversation history)
   - Active Projects (from workspace scan)
   - Key Technical Decisions (from past conversations)
   - User Corrections & Preferences
   - Environment Notes

Template:
```markdown
# MEMORY.md — Antigravity Dream Memory Index
<!-- Last consolidated: {ISO_TIMESTAMP} -->
<!-- Conversations scanned: {N} -->
<!-- Dream scope: project -->

## User Profile
- (populated from conversation history)

## Active Projects
### 1. {Project Name} (ACTIVE)
- **Location**: `{path}`
- **Stack**: {detected from config files}
- **Status**: {from conversation history or "New — needs context"}

## Key Technical Decisions
- (populated from conversation history)

## User Corrections & Preferences
- (populated from conversation history)

## Environment Notes
- **Workspace root**: `{path}`
```

#### CLAUDE.md (per project directory)
Read the project's config files and source structure to auto-populate:
1. Identify the project type (Java/Gradle, Node/npm, Python, etc.)
2. Read the main config file (build.gradle, package.json, etc.)
3. Summarize what the project IS and IS NOT
4. List critical files and rules

Template:
```markdown
# CLAUDE.md — {Project Name} Context
<!-- Auto-generated by dream bootstrap: {DATE} -->

## Project Identity
{One-line description based on config files}

## What This Project IS
- {Detected from source scan}

## What This Project IS NOT
- {Inferred boundaries}

## Critical Files
| File | Purpose |
|---|---|
| {detected files} | {purpose} |

## Rules
{Extracted from existing docs, READMEs, or conventions}

## Build
{Detected build commands}
```

#### USER_MEMORY.md (cross-project)
Only create if it doesn't exist at `~/.gemini/antigravity/USER_MEMORY.md`:
1. Scan all conversation history for user patterns
2. Detect preferences, coding style, common tools
3. Include the dream protocol instructions

**After bootstrap, proceed to Phase 1.**

### Phase 1: Orient
Read the current memory state:
- `MEMORY.md` in the workspace root (project-level)
- `CLAUDE.md` in each project directory (project context)
- `USER_MEMORY.md` in `~/.gemini/antigravity/` (user-level, cross-project)

Map what you currently "know" and identify the timestamp of last consolidation.
If any file was just bootstrapped in Phase 0, note it as `[NEW]` in the dream report.

### Phase 2: Gather Signal
Scan recent conversation transcripts for:
- **User corrections** — things the user told you that contradict your memory
- **Key decisions** — architectural choices, technology picks, design patterns agreed upon
- **New project state** — files created, deleted, renamed, or moved
- **Resolved issues** — bugs fixed, problems solved, questions answered
- **Changed dependencies** — version upgrades, new libraries, removed packages

Use the automation script to accelerate this:
```bash
python3 ~/.gemini/antigravity/skills/dream/scripts/dream_gather.py [scope]
```

### Phase 3: Consolidate & Prune
Apply these transformations to the memory files:

| Rule | Example |
|---|---|
| Delete contradictions | If MEMORY says "Sodium 0.5.8" but project now uses 0.5.13, update to 0.5.13 |
| Convert relative dates | "yesterday" → "2026-04-25" |
| Remove stale notes | References to deleted files, completed TODOs, resolved questions |
| Merge duplicates | If the same fact appears in multiple places, keep one authoritative entry |
| Archive completed work | Move completed project entries from "Active" to "Completed" |
| Update line counts | Ensure each memory file stays under 200 lines |

### Phase 4: Rebuild Index
Update the memory files:
1. Update the `<!-- Last consolidated: -->` timestamp
2. Update the `<!-- Conversations scanned: -->` count
3. Verify total line count is under 200
4. If over 200 lines, aggressively prune lowest-value entries

## Dream Scopes

| Command | Scope | Files Updated |
|---|---|---|
| `dream` or `dream #` | Current project | `MEMORY.md`, project `CLAUDE.md` |
| `dream user` | User-level | `USER_MEMORY.md` |
| `dream all` | Everything | All memory files |

## Memory File Locations

| File | Path | Max Lines | Purpose |
|---|---|---|---|
| MEMORY.md | `{workspace_root}/MEMORY.md` | 200 | Project workspace memory index |
| CLAUDE.md | `{project_dir}/CLAUDE.md` | 50 | Project-specific quick context |
| USER_MEMORY.md | `~/.gemini/antigravity/USER_MEMORY.md` | 50 | Cross-project user preferences |
